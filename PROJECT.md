# PROJECT.md
## Overview
`paren` is a Flutter app focused on travel-friendly currency workflows. It combines three related capabilities:
- Currency conversion with live exchange rates
- Favorite conversions for quick reuse
- Trip budgeting and expense tracking through reusable sheets
The main app entrypoint is `lib/main.dart`.
## Core Features
### Currency Conversion
- Convert between currencies using live exchange-rate data
- Refresh rates manually from the conversion screen
- Format values using locale-aware currency formatting
- Share conversion results from the app
### Favorites
- Save commonly used conversion pairs and amounts
- Reuse or remove favorites from the main workflow
- Persist favorites locally across launches
### Sheets / Trip Budgeting
- Create named sheets for trips or budgets
- Store entries with amounts and timestamps
- Keep separate source and target currencies per sheet
- Reorder, update, clear, and delete sheets and sheet entries
### Charts and History
- Fetch historical exchange-rate data for a selected currency pair
- Visualize trends with `fl_chart`
- Offer a simple prediction-style extension based on recent changes
### Customization
- Theme selection using a configurable app seed color
- Light, dark, and system theme behavior
- Localization through generated Flutter l10n files
- UI sizing and customization options stored in app settings
### Widgets and Platform Support
- Supports Android and iOS home widgets through `home_widget`
- Targets `android/`, `ios/`, `web/`, `macos/`, `linux/`, and `windows/`
## Architecture
### App Boot
`lib/main.dart` does the startup work:
- Initializes Flutter bindings
- Configures iOS widget app group support when relevant
- Instantiates the shared `Paren` GetX controller
- Loads saved settings before rendering the app
- Enables path-based URLs for web
- Builds `GetMaterialApp` with theme and locale driven by reactive state
### State Management
The app uses GetX throughout the project.
Primary state lives in `lib/providers/paren.dart`, which is responsible for:
- API client setup with `dio`
- Persisted settings and app preferences
- Currency list and selected conversion pair
- Current text input and conversion display state
- Favorites management
- Sheets and sheet-entry management
- Theme and locale updates
- Widget data updates for supported mobile platforms
### Screen Structure
- `lib/screens/home.dart`: Main shell that initializes app data and renders the responsive layout
- `lib/screens/home/conversion.dart`: Converter view, refresh behavior, formatted outputs, and sharing
- `lib/screens/home/sheets.dart`: Sheet listing and budget/trip management UI
- `lib/screens/home/customization.dart`: Appearance, locale, and related settings
On larger screens, the app presents a multi-pane layout. On smaller screens, it uses a paged experience.
### Reusable Components
The UI is split into reusable widgets under `lib/components/`, including:
- Header and navigation controls
- Calculator-style numeric input
- Currency selection controls
- Favorites and quick conversion widgets
- Budget planner and chart widgets
- Bottom-sheet forms for sheet creation and editing
## Data Model
The app uses JSON-serializable models under `lib/classes/`.
### `Currency`
Represents a currency and its live rate.
- `id`
- `name`
- `symbol`
- `rate`
### `FavoriteConversion`
Represents a saved conversion shortcut.
- `id`
- `fromCurrency`
- `toCurrency`
- `amount`
- `timestamp`
### `Sheet`
Represents a travel or budget sheet.
- `id`
- `name`
- `fromCurrency`
- `toCurrency`
- `createdAt`
- `updatedAt`
- `entries`
### `SheetEntry`
Represents an item inside a sheet.
- `id`
- `name`
- `createdAt`
- `updatedAt`
- `amount`
Generated serialization files live next to these models as `*.g.dart` and should not be edited manually.
## Data Sources and Persistence
- Exchange-rate and historical-rate data are fetched from the Frankfurter API via `https://api.frankfurter.dev/v1`
- App settings, favorites, currencies, and sheets are persisted locally with `shared_preferences`
- The app refreshes stale currency data instead of fetching on every launch
## Localization
Localization resources live in `lib/l10n/`.
Generated localization Dart files are derived from the `.arb` files and should not be edited by hand.
## Development Notes
- GetX reactive primitives such as `.obs` and `Obx(() => ...)` are the standard state pattern in this project
- `lib/classes/*.g.dart` files are generated by `json_serializable`
- `lib/l10n/app_localizations*.dart` files are generated localization output
- Widget preview annotations are present in parts of the UI for development previews
